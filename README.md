# drive-beagle

A small, polished, free/open desktop sync utility for Google-Drive-backed
folders, using `rclone` as the sync engine. Designed for developers who
want chosen Google Drive folders mirrored locally so coding agents
(Claude Code, Codex, …) can read and react to them.

* **Linux**: Flutter desktop app with file watching via `inotifywait`.
* **macOS**: same Flutter codebase, but launched as a **menu-bar agent**
  (`LSUIElement`) with file watching via `fswatch`.
* **CLI**: `drive-beagle` binary for headless control plus a stable,
  agent-facing change API (`changes`, `changes-since`, `last-sync`,
  `snapshot`, `diff`, `watch-events`, `ack`).
* Safe by default: dry-run by default, explicit bisync bootstrap,
  per-pair pause, debounced sync, single-flight per pair.

## Status

Early MVP. The architecture is documented in

## First-time setup

Flutter platform runner directories (`linux/`, `macos/Flutter/`, etc.) are
generated by Flutter on first build. After cloning, run:

```bash
flutter create --platforms=linux,macos --project-name drive_beagle .
```

Then **restore** our customized `macos/Runner/AppDelegate.swift` and
`macos/Runner/Info.plist` (Flutter overwrites them with templates):

```bash
git checkout macos/Runner/AppDelegate.swift macos/Runner/Info.plist
```

The `LSUIElement=YES` plist entry is what makes the app run as a menu-bar
agent without a dock icon.

## Prerequisites

* **rclone** ≥ v1.65 with `bisync` support
  ([install docs](https://rclone.org/install/)).
* **Dart** ≥ 3.4 / **Flutter** ≥ 3.22.
* **Linux**: `inotify-tools` (`pacman -S inotify-tools` on Arch).
* **macOS**: `fswatch` (`brew install fswatch`).

## Setup on Arch Linux

```bash
# 1. Install dependencies.
sudo pacman -S rclone inotify-tools
yay -S flutter   # or: install via your preferred method

# 2. Authenticate Google Drive (rclone handles OAuth).
rclone config
# → New remote → name: gdrive → type: drive → follow prompts

# 3. Build the CLI binary and the Flutter app.
flutter pub get
dart pub get -C packages/beagle_core
dart pub get -C packages/beagle_cli
dart compile exe packages/beagle_cli/bin/drive_beagle.dart -o build/drive-beagle

# 4. Configure your first sync pair.
mkdir -p ~/memory
./build/drive-beagle add \
    --name memory \
    --local-path ~/memory \
    --remote gdrive \
    --remote-path memory \
    --mode bidirectional

# 5. Run doctor.
./build/drive-beagle doctor

# 6. Bootstrap bisync (one-time, in the app or CLI).
./build/drive-beagle sync-now --pair memory --bootstrap

# 7. Start the app.
flutter run -d linux
# or: flutter build linux && ./build/linux/x64/release/bundle/drive_beagle
```

Optional: install the systemd-user unit at
`examples/drive-beagle.service` to autostart on login.

## Setup on macOS

```bash
# 1. Install dependencies.
brew install rclone fswatch

# 2. Configure rclone.
rclone config   # add a "gdrive" remote as above

# 3. Build the CLI and the menu-bar app.
flutter pub get
dart compile exe packages/beagle_cli/bin/drive_beagle.dart -o build/drive-beagle
flutter build macos --release
cp -r build/macos/Build/Products/Release/drive_beagle.app /Applications/drive-beagle.app

# 4. Configure your first sync pair (CLI works headless).
mkdir -p ~/Documents/memory
./build/drive-beagle add \
    --name memory \
    --local-path ~/Documents/memory \
    --remote gdrive \
    --remote-path memory \
    --mode bidirectional

# 5. Bootstrap.
./build/drive-beagle sync-now --pair memory --bootstrap

# 6. Install the launchd LaunchAgent for autostart.
cp examples/com.drivebeagle.agent.plist ~/Library/LaunchAgents/
launchctl bootstrap gui/$UID ~/Library/LaunchAgents/com.drivebeagle.agent.plist
```

The app marks itself as `LSUIElement` so it doesn't appear in the dock; look
for the cloud icon in your menu bar.

## Agent integration in 60 seconds

```bash
# What changed in `memory` that I haven't processed yet?
drive-beagle changes \
  --pair memory \
  --consumer claude-code \
  --unacked \
  --extensions .md \
  --format json

# After processing, advance your cursor:
drive-beagle ack --pair memory --consumer claude-code --cursor 173
```

Full agent API in [docs/agent-integration.md](docs/agent-integration.md).

## CLI cheatsheet

| Command                                                    | What it does |
|------------------------------------------------------------|--------------|
| `drive-beagle doctor`                                      | Validate dependencies, watch limits, remote reachability |
| `drive-beagle status`                                      | Show sync pairs and live state |
| `drive-beagle add --name … --local-path … --remote … …`    | Add a sync pair |
| `drive-beagle remove --pair <id>`                          | Remove a sync pair from config |
| `drive-beagle pause [--pair <id>]`                         | Pause sync globally or per pair |
| `drive-beagle resume [--pair <id>]`                        | Resume |
| `drive-beagle sync-now --pair <id> [--bootstrap]`          | Force a sync run (or bisync resync) |
| `drive-beagle dry-run --pair <id>`                         | Preview without writing |
| `drive-beagle logs [--pair <id>] [--follow]`               | Tail JSONL logs |
| `drive-beagle export-config` / `import-config`             | Round-trip the config |
| `drive-beagle changes --pair <id> --consumer <c> --unacked`| Authoritative changes for an agent |
| `drive-beagle changes-since --pair <id> --since <ts>`      | Time-based variant |
| `drive-beagle last-sync --pair <id>`                       | Last attempted/successful run metadata |
| `drive-beagle snapshot --pair <id> --side local|remote`    | Snapshot |
| `drive-beagle diff --pair <id> --against local|remote`     | Diff snapshots |
| `drive-beagle watch-events --pair <id>`                    | Live JSONL of tentative watcher events |
| `drive-beagle ack --pair <id> --consumer <c> --cursor <n>` | Advance a cursor |

Exit codes are stable — see `packages/beagle_cli/lib/cli_runner.dart`.

## Safety defaults

* New pairs default to `dry_run` mode. Switch to `bidirectional` only after
  reviewing what would change.
* Bidirectional pairs require an explicit one-time `--bootstrap` (rclone
  bisync `--resync`). Without it, runs fail with `BISYNC_NEEDS_RESYNC`.
* Conflicts default to "keep both with `.local`/`.remote` suffix" — nothing
  is overwritten silently.
* Sync runs are strictly serialized per pair and use `rclone --resilient
  --recover --retries 3`.
* Hidden files, VCS dirs, `node_modules`, common build dirs, editor
  swap/backup files are excluded by default.

## Troubleshooting

* `RCLONE_MISSING` / `RCLONE_VERSION_TOO_OLD` — install or upgrade rclone.
* `WATCH_LIMIT_LOW` (Linux) — bump `fs.inotify.max_user_watches`:
  ```
  echo "fs.inotify.max_user_watches=524288" | sudo tee /etc/sysctl.d/99-drive-beagle.conf
  sudo sysctl --system
  ```
* `BISYNC_NEEDS_RESYNC` — run `drive-beagle sync-now --pair <id> --bootstrap`.
* `CURSOR_MISMATCH` — your cursor is ahead of the local journal. Either
  re-discover via `drive-beagle changes --pair <id>` or pass
  `drive-beagle doctor --rebuild-index` if you suspect journal damage.
